# 报告生成文件关系说明（背景扫描 / 日报扫描）

本文档梳理仓库内与“背景扫描.md”“日报扫描.md”两类报告直接相关的代码文件、调用链路与产出路径，便于定位与维护。

## 1. 概要
- 报告并非直接写入仓库根目录的"背景扫描.md / 日报扫描.md"，而是由生成器写入 `reports/` 目录中的带时间戳的 Markdown 文件。
- 两类报告共用同一套数据引擎与指标计算模块，仅模板/排版与入口选择不同。
- **新增**: 库存周期验证机制，所有商品趋势判断必须通过技术面+库存周期双重验证，确保投资决策的基本面支撑。

## 2. 直接相关文件
- 入口与编排
  - `run_na_filling.py`：命令行入口，选择生成“背景扫描报告/日表报告/二者皆有”，内部调用报告生成器。

- 报告生成核心
  - `src/datasource/generators/report_generator.py`：报告生成器
    - `generate_background_scan_report(days)`：生成“背景扫描”版（使用内置模板 `_get_background_scan_template()`）
    - `generate_daily_table_report(days)`：生成“日表”版（使用外部模板）
    - `generate_both_reports(days)`：并行生成两份报告
    - 返回值字典键：`background_scan`（背景扫描）与 `daily_table`（日表），统一报告会兼容旧版中文键名。
    - 输出写入目录：`reports/`
  - `templates/daily_report_template.md`：日表报告模板（占位符由生成器填充）

- 数据汇总与格式化
  - `src/datasource/engines/data_engine.py`：数据引擎
    - 并发聚合：A 股指数技术面、债券收益率、资金流向、普林格六阶段
    - `get_formatted_market_data()` 将原始数据转换为报告可填充的占位字段

- 指标计算与分析
  - `src/datasource/calculators/technical_indicators.py`：技术指标/趋势评分 + **库存周期验证计算**
  - `src/datasource/calculators/bond_calculator.py`：债券收益率（ETF 代理估算）
  - `src/datasource/calculators/fund_flow_calculator.py`：北/南向资金估算与聚合
  - `src/datasource/calculators/pring_analyzer.py`：普林格"六阶段"判定与建议 + **库存周期阶段识别**

- 数据获取与故障转移
  - `src/datasource/manager.py`：数据源总管（统一 API、主备源与故障转移）
  - `src/datasource/adapters/akshare_adapter.py`：AKShare 适配器（限流/缓存/重试）
  - `src/datasource/adapters/tushare_adapter.py`：TuShare 适配器（Token、Pro 接口或降级）
  - `src/datasource/models/base.py`：抽象接口与 `DataResponse`/`DataSourceConfig`

- 通用支撑
  - `src/datasource/utils/rate_limiter.py`：异步速率限制
  - `src/datasource/utils/retry.py`：异步重试（指数退避）
  - `src/datasource/cache/memory_cache.py`：内存缓存与键生成

- 配置（影响取数）
  - `.env.example`（以及本地 `.env`）：`TUSHARE_TOKEN`、速率限制与缓存 TTL，供 `manager/adapters` 读取

## 3. 不直接参与报告生成的相关文件（参考）
- 根目录文档：`背景扫描.md`、`日报扫描.md`、`日表样例.md` → 作为样例/说明，不在程序生成链路中被读取或覆盖。
- 示例/测试脚本：`market_scan_data.py`、`example.py`、`simple_test.py`、`test_*.py` → 用于演示或验证，不是正式生成路径。

## 4. 产出路径与文件命名
- 背景扫描报告：`reports/background_scan_report_<YYYYmmdd_HHMMSS>.md`
- 日表报告：`reports/daily_table_report_<YYYYmmdd_HHMMSS>.md`

> 如需直接在仓库根目录生成固定文件名（例如“背景扫描.md”“日报扫描.md”），可在 `report_generator.py` 中：
> - 将 `self.output_dir` 改为仓库根目录（例如 `Path(__file__).resolve().parents[3]`）
> - 将文件名固定为 `"背景扫描.md"` 或 `"日报扫描.md"`
> 或者在 `run_na_filling.py` 生成后追加重命名/复制逻辑。

## 5. 调用链路速览（含库存周期验证）
1) 入口：`run_na_filling.py`（解析参数、选择生成类型）
2) 生成器：`ReportGenerator.generate_*()`（加载模板 → 获取/格式化数据 → 写入 MD 文件）
3) 引擎：`MarketDataEngine.get_formatted_market_data()`（并发取数与汇总）
4) 计算：`calculators/*`（技术指标、债券、资金流、普林格 + **库存周期验证**）
   - **新增**: PPI/CPI/PMI数据获取与库存周期阶段判断
  - **新增**: 商品Bullish双重验证（技术面35% + 库存周期65%）
5) 取数：`DataSourceManager` → `adapters/*`（缓存/限流/重试/故障转移）
6) 配置：读取 `.env`（Token、限流、缓存）

## 6. 运行方式（示例）
```bash
# 生成两类报告到 reports/
python run_na_filling.py --days 30 --type both

# 仅生成背景扫描报告
python run_na_filling.py --days 30 --type background

# 仅生成日表报告
python run_na_filling.py --days 30 --type daily
```

## 7. 库存周期验证框架集成 (2025.9.10 新增)

### 验证机制说明
- **技术面评分 (35%权重)**: 基于价格趋势、成交量确认、BDI指数等传统技术指标
- **库存周期评分 (60%权重)**: 基于PPI、PMI、CPI等宏观基本面数据验证
- **综合评分标准**: ≥70分判定Bullish，≤30分判定Bearish，中间为N/A

### 相关文档参考
- **[库存周期验证标准.md](库存周期验证标准.md)**: 完整的验证框架和评分标准
- **[技术指标计算说明.md](技术指标计算说明.md)**: 包含库存周期验证的具体实现方法
- **背景扫描.md / 日报.md**: 已更新模板要求强制进行库存周期验证

### 实战验证案例
- **2025.9.10验证结果**: 原技术面显示商品偏强，但库存周期验证显示PPI深度通缩(-3.6%)、PMI收缩(49.4%)，综合评分约22.6分→Bearish，成功避免配置风险
