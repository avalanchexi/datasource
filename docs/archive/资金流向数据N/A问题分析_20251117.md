```python
class FundFlowData(BaseModel):
    @staticmethod
    def _parse_amount(value: Optional[Any]) -> Optional[float]:
        if value is None or value == '':
            return None

        if isinstance(value, (int, float)):
            return float(value)

        if isinstance(value, str):
            text = value.strip()
            if not text:
                return None

            # 解析 "1300亿", "1.3万亿", "500" 等格式
            multiplier = 1.0
            if '万亿' in text:
                multiplier = 10000.0
            elif '千亿' in text:
                multiplier = 1000.0
            elif '亿' in text:
                multiplier = 1.0

            match = re.search(r'[-+]?\d+(?:\.\d+)?', text)
            if not match:  # ← "波动"、"净流入"等无数字，返回None
                return None

            amount = float(match.group()) * multiplier
            return amount
```

## 三、经验教训

### 3.1 数据格式要求

**FundFlowData._parse_amount() 支持的格式**:

✅ **支持**:
- 纯数字: `"1300"` → 1300.0
- 数字+单位: `"1300亿"` → 1300.0
- 数字+单位: `"1.3万亿"` → 13000.0
- 负数: `"-100"` → -100.0
- 浮点数: `"46.2"` → 46.2

❌ **不支持**:
- 纯文本: `"波动"` → None
- 趋势词: `"净流入"` → None
- 描述性文本: `"高位盘整"` → None
- 空字符串: `""` → None

### 3.2 WebSearch数据收集规范

**正确做法**:
1. ✅ **recent_5d** / **total_120d** 字段必须填写**可解析的数值**
2. ✅ 描述性信息放在 **note** 字段中
3. ✅ 单位信息放在 **unit** 字段中（可选，但建议填写）
4. ✅ 趋势判断由系统根据数值自动推断（正数=流入，负数=流出）

**错误做法**:
1. ❌ 在 recent_5d/total_120d 中填写"波动"、"净流入"等文字
2. ❌ 在金额字段中混入描述文本（除非包含明确数字）
3. ❌ 依赖人工在报告生成阶段手动填写数据

### 3.3 数据注入验证

**注入脚本输出示例**:

```bash

  [OK] etf: recent_5d=46.2 total_120d=46900.0 source=MCP WebSearch实时获取
```

**验证要点**:
- ✅ 所有项都显示 `[OK]` 而非 `[WARN]`
- ✅ recent_5d 和 total_120d 均为浮点数，非 None
- ✅ 注入项数应为 4（北向+南向+ETF+融资融券）

```python
# validate_websearch_data.py
def validate_fund_flow(websearch_json):
    errors = []
    for key, data in websearch_json['fund_flow'].items():
        # 检查必需字段
        if 'recent_5d' not in data or 'total_120d' not in data:
            errors.append(f"{key}: 缺少必需字段")

        # 检查是否可解析为数值
        from datasource.models.market_data_contract import FundFlowData
        if FundFlowData._parse_amount(data.get('recent_5d')) is None:
            errors.append(f"{key}.recent_5d='{data.get('recent_5d')}' 无法解析为数值")

        if FundFlowData._parse_amount(data.get('total_120d')) is None:
            errors.append(f"{key}.total_120d='{data.get('total_120d')}' 无法解析为数值")

    return errors
```

## 五、总结

### 问题本质
AI在执行WebSearch时，将资金流向的**趋势描述**（"波动"、"净流入"）误填入了应该填写**数值**的字段（recent_5d），导致数据解析器无法识别，最终在报告中显示为N/A。

### 核心解决方案
1. **数据源头规范化**: WebSearch结果JSON中，金额字段必须填写可解析的数值
2. **数据验证前置**: 在注入阶段检查所有关键字段的解析结果
3. **自动化检测**: 报告生成后自动检查是否存在N/A数据

### 流程优化建议
1. 为WebSearch任务提供明确的数据格式模板
2. 在数据注入脚本中增加更严格的验证逻辑
3. 在报告生成后自动执行质量检查（文件大小、章节数、N/A数量）
