#### 2. 移动平均线计算（V2.0配置化）

```python
def calculate_moving_averages(df):
    """计算移动平均线 - 基于统一配置"""
    from datasource.config import TECHNICAL_PARAMS
  
    df = df.sort_values('trade_date')  # 确保按时间排序
  
    # 从配置获取MA参数
    ma_periods = TECHNICAL_PARAMS['moving_averages']
    ma20_period = ma_periods['short'][2]   # 20
    ma50_period = ma_periods['medium'][0]  # 50  
    ma200_period = ma_periods['long'][0]   # 200
  
    # 计算各期移动平均线（基于配置）
    df[f'MA{ma20_period}'] = df['close'].rolling(window=ma20_period).mean()
    df[f'MA{ma50_period}'] = df['close'].rolling(window=ma50_period).mean()
    df[f'MA{ma200_period}'] = df['close'].rolling(window=ma200_period).mean()
  
    return df
```

#### 4. 斜率计算

```python
def calculate_slope(df, ma_column, periods=5):
    """计算移动平均线斜率"""
    if len(df) < periods + 20:  # 确保有足够数据
        return None
      
    current_ma = df[ma_column].iloc[-1]
    prev_ma = df[ma_column].iloc[-periods-1]
  
    if current_ma and prev_ma:
        slope = current_ma - prev_ma
        return "↑" if slope > 0 else ("↓" if slope < 0 else "→")
    return None
```

#### 5. 年化波动率计算

```python
def calculate_volatility(df, periods=30):
    """计算年化波动率"""
    if len(df) < periods:
        return None
      
    returns = df['close'].pct_change().dropna()
    if len(returns) >= periods:
        vol_30d = returns.tail(periods).std() * np.sqrt(252) * 100
        return vol_30d
    return None
```

```python
def calculate_trend_score(current_price, ret_30d, ma50, ma200, ma20_slope):
    """计算趋势评分 - 基于配置参数"""
    from datasource.config import TECHNICAL_PARAMS
  
    scoring_params = TECHNICAL_PARAMS['trend_scoring']
    threshold = scoring_params['price_change_threshold']  # 1.0%
    score_range = scoring_params['score_range']  # [-2, 2]
  
    score = 0
  
    # 近30日收益基于配置阈值
    if ret_30d and ret_30d >= threshold:
        score += 1
    elif ret_30d and ret_30d <= -threshold:
        score -= 1
      
    # 收盘高于MA50 (+1分)
    if ma50 and current_price > ma50:
        score += 1
    elif ma50 and current_price < ma50:
        score -= 1
      
    # MA50 > MA200 (+1分)
    if ma50 and ma200:
        if ma50 > ma200:
            score += 1
        else:
            score -= 1
          
    # MA20斜率为正 (+1分)
    if ma20_slope == "↑":
        score += 1
    elif ma20_slope == "↓":
        score -= 1
  
    # 截断到 [-2, +2] 范围
    score = max(-2, min(2, score))
  
    # 生成标签
    if score >= 1:
        label = "牛"
    elif score <= -1:
        label = "熊"
    else:
        label = "中性"
      
    return score, f"{label}（评分{score:+d}）"
```

```python
        elif raw_materials_inventory > 50 and finished_goods_inventory > 50:

        'finished_goods': finished_goods_inventory,
```

```python
def verify_commodity_bullish(price_data, ppi_data, pmi_data, bdi_data):
    """商品Bullish状态双重验证"""
  
    # 技术面评分 (30%权重)
    technical_score = calculate_technical_score(price_data)
  
    # 库存周期评分 (60%权重) 
    inventory_score = 0
  
    # PPI环比连续为正验证 (25分)
    ppi_info = get_ppi_data()
    if ppi_info['consecutive_months'] >= 2:
        inventory_score += 25
  
    # PPI同比改善验证 (15分)
    if ppi_info['latest_yoy'] > -0.5:  # 降幅收窄
        inventory_score += 15
  
    # PMI库存去化验证 (20分)
    pmi_inventory = analyze_pmi_inventory(pmi_data)
    if pmi_inventory and pmi_inventory['stage'] == "去库存阶段":
        inventory_score += 20
  
    # 综合评分
    total_score = technical_score * 0.35 + inventory_score * 0.65
  
    if total_score >= 70:
        verdict = "Bullish"
    elif total_score <= 30:
        verdict = "Bearish"
    else:
        verdict = "N/A(验证不充分)"
  
    return {
        'technical_score': technical_score,
        'inventory_score': inventory_score,
        'total_score': total_score,
        'verdict': verdict,
        'verification_details': {
            'ppi_consecutive': ppi_info['consecutive_months'],
            'ppi_yoy_trend': ppi_info['latest_yoy'],
            'inventory_stage': pmi_inventory['stage'] if pmi_inventory else "数据不足"
        }
    }
```

# 库存周期验证标准

**创建日期**: 2025-09-10
**版本**: v1.2.0
**适用范围**: 商品趋势判断、普林格六阶段分析、资产配置决策
**V1.2更新**: 集成最新统计数据获取和动态日期更新功能

## 📋 执行摘要

基于2025年9月10日的实战验证，发现仅依赖技术分析判断商品趋势存在重大风险。本标准建立了"技术面+库存周期"双重验证机制，要求商品Bullish判断必须同时通过价格技术面和宏观基本面验证，避免在库存周期不支持时做出错误的配置决策。

**核心原则**: 技术面30% + 库存周期70% = 综合判断，≥70分判定Bullish，≤30分判定Bearish，中间为N/A。

- `scripts/utility/calculate_na_data.py`：集成最新库存周期验证的NA数据计算
- `generate_report_simple.py`：生成包含最新宏观数据的完整分析报告

## 🎯 验证标准概览

### 商品Bullish判定的必要条件

**必须全部满足以下条件**：

| 验证维度               | 权重 | 核心指标        | 判定标准                      | 数据来源        |
| ---------------------- | ---- | --------------- | ----------------------------- | --------------- |
| **技术面验证**   | 30%  | 价格趋势+成交量 | 价格>MA50>MA200，30日涨幅≥3% | AKShare/TuShare |
| **库存周期验证** | 70%  | PPI+PMI+BDI     | PPI环比连续2月+，PMI库存去化  | 国家统计局      |

**关键门槛**: 库存周期评分必须≥40分（满分70分），否则无论技术面多强都判定为N/A或Bearish。

## 🔍 详细验证框架

### 一、库存周期阶段判断矩阵

| PPI同比 | PMI库存 | 库存周期阶段 | 商品趋势倾向   | 配置建议       |
| ------- | ------- | ------------ | -------------- | -------------- |
| ↑      | ↓      | 主动补库存   | **强牛** | 重配商品15-25% |
| ↑      | ↑      | 被动补库存   | **偏牛** | 适配商品10-15% |
| ↓      | ↓      | 主动去库存   | **中性** | 观望商品0-5%   |
| ↓      | ↑      | 被动去库存   | **熊**   | 避开商品0%     |

### 二、评分体系详解

#### 技术面评分（30%权重，满分30分）

**价格趋势评分（25分）**：

- 收盘价>MA50 且 MA50>MA200：+15分
- 30日涨跌幅≥5%：+10分，1-5%：+5分，<1%：+0分
- MA20斜率向上：+5分，持平：+0分，向下：-5分

**成交量确认（15分）**：

- 成交量放大配合价格上涨：+10分
- BDI指数30日涨幅>5%：+5分

#### 库存周期评分（60%权重，满分60分）

**PPI通胀验证（25分）**：

- PPI环比连续2个月为正：+15分
- PPI同比降幅收窄至-0.5%以内：+10分

**PMI库存验证（20分）**：

- PMI原材料库存<50且连续下降：+10分
- PMI产成品库存<50：+10分

**需求验证（15分）**：

- 工业增加值同比回升：+5分
- CPI环比企稳或上行：+5分
- BDI指数上升：+5分

## 📊 实战验证案例

### 案例1: 2025年9月10日商品趋势验证

**背景**: 基于技术面，原油+5.85%、铜+2.95%，南华商品指数上涨，似乎显示商品转强

#### 技术面评分: 31/35分

- ✅ 商品价格短期上涨
- ✅ BDI指数9月反弹至1979点
- ❌ 缺乏综合商品指数数据确认

#### 库存周期评分: 18/65分

- ❌ PPI 7月同比-3.6%，环比-0.2%（深度通缩）
- ❌ PMI 8月49.4%，低于荣枯线（制造业收缩）
- ❌ CPI 7月同比0.0%（内需疲弱）
- ✅ BDI指数反弹（需求边际改善）

#### 综合判断

- **综合评分**: 31×35% + 18×65% ≈ **22.6分**
- **最终判断**: **Bearish**
- **阶段判断**: 被动去库存，第Ⅱ阶段（复苏早期）
- **配置调整**: 商品权重从15-20%降至0-5%

#### 验证结果价值

避免了基于技术面误判的重配商品风险，正确识别当前仍处于周期底部。

## 🚨 关键风险警示

### 常见误判风险

1. **纯技术面误导**: 商品价格技术性反弹不等于趋势反转
2. **单一指标依赖**: 仅看BDI或南华指数无法覆盖完整周期
3. **数据滞后陷阱**: 宏观数据滞后可能错过转折点
4. **政策预期干扰**: 强刺激预期可能造成虚假信号

### 风险控制措施

1. **双重验证机制**: 技术面+基本面必须同时确认
2. **评分门槛设置**: 库存周期评分<35分强制判N/A
3. **阶段性复核**: 每月复核库存周期位置
4. **应急调整机制**: 新数据发布后24小时内重新评估

# Pring六阶段分析增强V2.1

**版本**: v2.1.0
**更新日期**: 2025-09-11
**核心改进**: 商品信号集成库存周期矫正，避免纯技术面误判风险

## 📋 V2.1优化摘要

原Pring六阶段分析仅基于技术面（价格>MA200且MA50上穿MA200）判断商品信号，存在重大风险。经过2025年9月10日实战验证，发现仅依赖技术分析判断商品趋势可能导致错误的资产配置决策。

**核心问题**: 商品价格短期技术性反弹不等于趋势反转，需要宏观基本面确认。

**解决方案**: 集成"技术面30% + 库存周期70%"的双重验证机制。

## 🎯 V2.1增强特性

### 1. 库存周期矫正算法

```python
# 核心算法
综合评分 = 技术面评分 * 0.35 + 库存周期评分 * 0.65

# 判定标准
if 综合评分 >= 70: 商品信号 = Bullish
elif 综合评分 <= 30: 商品信号 = Bearish
else: 商品信号 = Neutral
```

```python
async def determine_commodity_signal_with_correction(self, start_date: str, end_date: str) -> AssetSignal:
    """基于技术面+库存周期矫正的商品信号判定"""
  
    # 第一步：获取技术面信号
    technical_score = await self.calculate_commodity_technical_score(start_date, end_date)
  
    # 第二步：获取宏观数据并计算库存周期评分
    macro_data = await self.get_macro_economic_data()
    cycle_analysis = self.calculate_inventory_cycle_score(macro_data)
  
    # 第三步：综合评分
    total_score = (
        technical_score * self.cycle_correction_weights["technical_weight"] + 
        cycle_analysis["fundamental_score"] * self.cycle_correction_weights["fundamental_weight"]
    )
  
    # 第四步：基于评分判定信号
    if total_score >= 70:
        return AssetSignal.BULLISH
    elif total_score <= 30:
        return AssetSignal.BEARISH
    else:
        return AssetSignal.NEUTRAL
```

```python
    """商品Bullish双重验证 (技术面30% + 库存周期70%)"""

    # PPI环比连续为正验证 (25分)
    ppi_info = analyze_ppi_trend(ppi_data)
    if ppi_info['consecutive_months'] >= 2:
        inventory_score += 25
  
    # 综合评分
    total_score = technical_score * 0.35 + inventory_score * 0.65
  
    if total_score >= 70:
        verdict = "Bullish"
    elif total_score <= 30:
        verdict = "Bearish" 
    else:
        verdict = "N/A(验证不充分)"
  
    return {
        'technical_score': technical_score,
        'inventory_score': inventory_score, 
        'total_score': total_score,
        'verdict': verdict
    }
```

```python
async def collect_macro_data(manager, config):
    """收集宏观经济数据用于库存周期验证"""
    results = {}
  
    try:

        ppi_data = await manager.get_macro_data('ppi')

            results['ppi'] = {
                'data': ppi_data.data,
                'source': ppi_data.source,
```

# 报告生成文件关系说明

本文档梳理仓库内与"背景扫描.md""日报扫描.md"两类报告直接相关的代码文件、调用链路与产出路径，便于定位与维护。

## 1. 概要

- 报告并非直接写入仓库根目录的"背景扫描.md / 日报扫描.md"，而是由生成器写入 `reports/` 目录中的带时间戳的 Markdown 文件。
- 两类报告共用同一套数据引擎与指标计算模块，仅模板/排版与入口选择不同。
- **新增**: 库存周期验证机制，所有商品趋势判断必须通过技术面+库存周期双重验证，确保投资决策的基本面支撑。

## 2. 直接相关文件

- 入口与编排

  - `run_na_filling.py`：命令行入口，选择生成"背景扫描报告/日表报告/二者皆有"，内部调用报告生成器。
- 报告生成核心

  - `src/datasource/generators/report_generator.py`：报告生成器
    - `generate_background_scan_report(days)`：生成"背景扫描"版（使用内置模板 `_get_background_scan_template()`）
    - `generate_daily_table_report(days)`：生成"日表"版（使用外部模板）
    - 输出写入目录：`reports/`
  - `templates/daily_report_template.md`：日表报告模板（占位符由生成器填充）
- 数据汇总与格式化

  - `src/datasource/engines/data_engine.py`：数据引擎
    - 并发聚合：A 股指数技术面、债券收益率、资金流向、普林格六阶段
    - `get_formatted_market_data()` 将原始数据转换为报告可填充的占位字段
- 指标计算与分析

  - `src/datasource/calculators/technical_indicators.py`：技术指标/趋势评分 + **库存周期验证计算**
  - `src/datasource/calculators/bond_calculator.py`：债券收益率（ETF 代理估算）
  - `src/datasource/calculators/fund_flow_calculator.py`：北/南向资金估算与聚合
  - `src/datasource/calculators/pring_analyzer.py`：普林格"六阶段"判定与建议 + **库存周期阶段识别**

## 3. 调用链路速览（含库存周期验证）

1) 入口：`run_na_filling.py`（解析参数、选择生成类型）
2) 生成器：`ReportGenerator.generate_*()`（加载模板 → 获取/格式化数据 → 写入 MD 文件）
3) 引擎：`MarketDataEngine.get_formatted_market_data()`（并发取数与汇总）
4) 计算：`calculators/*`（技术指标、债券、资金流、普林格 + **库存周期验证**）
   - **新增**: PPI/CPI/PMI数据获取与库存周期阶段判断
   - **新增**: 商品Bullish双重验证（技术面30% + 库存周期70%）
5) 取数：`DataSourceManager` → `adapters/*`（缓存/限流/重试/故障转移）
6) 配置：读取 `.env`（Token、限流、缓存）

## 4. 库存周期验证框架集成 (2025.9.10 新增，V1.2优化)

### 验证机制说明

- **技术面评分 (30%权重)**: 基于价格趋势、成交量确认、BDI指数等传统技术指标
- **库存周期评分 (70%权重)**: 基于PPI、PMI、CPI等宏观基本面数据验证
- **综合评分标准**: ≥70分判定Bullish，≤30分判定Bearish，中间为N/A

### V1.2版本技术改进

- ✅ **动态数据获取**: 基于 `datetime.now()`实时获取当前日期，确保数据时效性
- ✅ **智能提示系统**: 自动生成最新统计数据获取提示词模板
- ✅ **数据源验证**: 自动确认使用国家统计局最新发布版本
- ✅ **实时计算引擎**: 基于最新数据重新计算库存周期阶段和商品评分
- ✅ **集成报告生成**: 将最新宏观数据无缝集成到技术分析报告中

- **2025.9.10验证结果**: 原技术面显示商品偏强，但库存周期验证显示PPI深度通缩(-3.6%)、PMI收缩(49.4%)，综合评分27分→Bearish，成功避免配置风险
