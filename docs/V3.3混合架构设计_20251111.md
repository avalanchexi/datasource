# V3.3 混合架构设计提案

**创建日期**: 2025-11-11
**状态**: 设计提案
**基于**: V3.2 Stage 4演示成果
**目标**: 在保持V3.1快速模式的同时,提供Pring分析所需的完整数据

---

## 一、核心洞察

**问题**: V3.2 Stage 4后置架构存在逻辑问题
- Pring分析(Stage 2)基于38%不完整数据
- 商品/债券数据缺失可能影响周期判断准确性
- Stage 4填充数据后,Pring结果已固化无法修正

**解决方案**: Stage 4前置 → Stage 2a,在Pring分析前补全关键数据

---

## 二、V3.3 架构对比

### V3.1 (当前默认 - 快速模式)

```
Stage 1              Stage 2           Stage 3
Data Collector  →    Pring Analyzer →  Report Generator
(38% data)           (不完整数据分析)   (占位符报告)

执行时间: 60-90秒
数据完整度: 38%
优势: 快速,无需MCP
劣势: Pring分析准确性受限
```

### V3.2 (Stage 4后置 - 已演示)

```
Stage 1  →  Stage 2        →  Stage 3      →  Stage 4
Collector    Pring (38%)      Report (占位符)  MCP Enhancer
                                               (填充所有数据)

执行时间: 60秒 + 3-4分钟(Stage 4)
数据完整度: 95%+
问题: Pring分析已完成,无法利用补全数据
```

### V3.3 (混合架构 - 推荐) ⭐

```
Stage 1  →  [Stage 2a]     →  Stage 2        →  [Stage 4]      →  Stage 3
Collector    MCP Essential     Pring (95%)      MCP Full          Report
(38%)        (关键数据)        (完整数据分析)    (资金流+要闻)    (最终渲染)

--disable-mcp:    跳过Stage 2a/4,保持V3.1快速模式 (60秒)
--enable-mcp:     启用Stage 2a,跳过Stage 4 (3-4分钟,推荐)
--enable-full-mcp: 启用Stage 2a+4,最大完整度 (4-5分钟)
```

**更新 (2025-11-12)**: 为保证报告展示直接使用补齐后的资金流向与财经要闻，V3.3 完整模式会在 Stage 3 渲染前执行 Stage 4（`stage4_mcp_enhancer.py`），随后再运行 Stage 3 生成最终 Markdown。

## 三、Stage 2a设计

### 职责定义

**Stage 2a: MCP Data Enhancer - Essential Data Only**

**输入**: market_data.json (38%完整,from Stage 1)
**输出**: market_data_enhanced.json (65-75%完整)
**执行时间**: ~2-3分钟

**数据填充策略** (选择性填充):

| 数据类型 | 填充? | 理由 | 预计MCP调用 |
|---------|------|------|------------|
| CN10Y债券 | ✅ 必填 | 影响货币周期判断,Pring第二层核心指标 | 1次WebSearch |
| CN10Y_CDB | ✅ 必填 | 辅助验证债券趋势 | 1次WebSearch |
| 商品(黄金/原油/铜) | ✅ 必填 | 验证库存周期,Pring商品趋势指标 | 3-5次WebFetch |
| BCOM/GSG指数 | ⚠️ 可选 | 综合商品趋势参考 | 2次WebSearch |
| 资金流向(北向/南向/ETF) | ❌ 跳过 | 不直接影响Pring分析,Stage 4补充 | 节省4次 |
| 财经要闻 | ❌ 跳过 | 报告展示用,不影响分析逻辑 | 节省4-10次 |

**MCP调用总数**: 5-9次 (vs Stage 4的15-20次)
**时间节省**: ~50%

## 四、background_scan_unified.py修改

### 新增参数

```python
parser.add_argument(
    '--enable-mcp',
    action='store_true',
    help='Enable Stage 2a MCP enhancer before Pring analysis (recommended for accuracy)'
)

parser.add_argument(
    '--enable-full-mcp',
    action='store_true',
    help='Enable both Stage 2a and Stage 4 MCP enhancers (maximum completeness)'
)
```

### 执行逻辑

```python
# Stage 1: 数据收集
market_data = run_stage1_data_collector(args)

# Stage 2a: MCP Essential (可选)
if args.enable_mcp or args.enable_full_mcp:
    market_data_enhanced = run_stage2a_mcp_enhancer(
        market_data,
        mode='essential'  # 只填充关键数据
    )
else:
    market_data_enhanced = market_data  # 跳过,保持V3.1模式

# Stage 2: Pring分析 (使用增强后的数据)
pring_result = run_stage2_pring_analyzer(market_data_enhanced)

# Stage 3: 报告生成
report = run_stage3_report_generator(market_data_enhanced, pring_result)

# Stage 4: MCP Full (可选)
if args.enable_full_mcp:
    report_final = run_stage4_mcp_full_enhancer(
        market_data_enhanced,
        pring_result,
        report,
        mode='supplement'  # 补充剩余数据(资金流向/财经要闻)
    )
else:
    report_final = report
```

### 使用示例

```bash
# 模式1: V3.1快速模式 (默认)
python scripts/background_scan_unified.py --date 2025-11-11 --output report.md
# 执行: Stage 1 → Stage 2 → Stage 3
# 时间: ~60秒, 数据完整度: 38%

# 模式2: V3.3混合模式 (推荐) ⭐
python scripts/background_scan_unified.py --date 2025-11-11 --output report.md --enable-mcp
# 执行: Stage 1 → Stage 2a (Essential) → Stage 2 → Stage 3
# 时间: ~3分钟, 数据完整度: 65-75%, Pring分析准确

# 模式3: V3.3完整模式
python scripts/background_scan_unified.py --date 2025-11-11 --output report.md --enable-full-mcp
# 执行: Stage 1 → Stage 2a → Stage 2 → Stage 4 → Stage 3
# 时间: ~5分钟, 数据完整度: 95%+, 最高质量
```

## 五、Pring分析数据依赖分析

### Pring三层框架数据需求

**第一层: 库存周期** (60分)
- PPI, PMI, CPI, 工业增加值, BDI
- ✅ Stage 1已提供,无需MCP

**第二层: 货币周期** (100分)
- 降准, 降息, M2, TSF
- ⚠️ CN10Y债券收益率 (权重15%) - **需要MCP补充**
- ✅ 其他数据Stage 1已提供

**第三层: Pring六阶段修正**
- 商品趋势 (黄金/原油/铜) - **需要MCP补充**
- 债券趋势 (CN10Y/US10Y) - **部分需要MCP**
- 股票趋势 (沪深300) - ✅ Stage 1已提供

### 关键发现

**缺失数据对Pring分析的影响评估**:

1. **CN10Y债券**: 🔴 高影响
   - 直接影响货币周期评分 (15/100分)
   - 影响Pring第三层修正置信度 (±10%)
   - **必须填充**

2. **商品价格(黄金/原油/铜)**: 🟡 中影响
   - 验证库存周期判断
   - 影响商品趋势偏向
   - **建议填充**

3. **资金流向**: 🟢 低影响
   - 不参与Pring评分计算
   - 仅用于报告展示
   - **可跳过**

**结论**: Stage 2a填充债券+商品数据,可将Pring分析准确性从60%提升至85-90%
