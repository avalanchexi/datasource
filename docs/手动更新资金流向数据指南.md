# 手动更新资金流向数据指南

**版本**: V4.0
**更新日期**: 2026-05-19
**目的**: 当Stage2自动获取失败时，手动补充资金流向数据

---

## 前置条件

确保已完成Stage1和Stage2基础流程，生成了 `data/${DATE}_market_data_stage2.json`。

---

## 步骤1: 获取实时数据

### 1.1 北向资金 (Northbound)

**数据源**: [东方财富网-沪深港通](https://data.eastmoney.com/hsgt/)

**需要获取的数据**:
- 近5日净流入金额（亿元）- 必须是数字
- 近120日累计净流入金额（亿元）- 必须是数字
- 流向趋势（流入/流出）

### 1.2 南向资金 (Southbound)

**数据源**: [同花顺-港股通](https://data.10jqka.com.cn/hgt/ggta/)

**需要获取的数据**:
- 近5日净流入金额（亿元）
- 近120日累计净流入金额（亿元）
- 流向趋势

### 1.3 ETF资金流 (ETF)

**数据源**: 优先使用可证明全市场窗口的结构化入口，例如 [东方财富ETF数据](https://data.eastmoney.com/etf/)；新闻、季度报告或 `fund.eastmoney.com` 资讯页只能作为估算依据。

**需要获取的数据**:
- 近5日ETF净申购金额（亿元）
- 近120日累计净申购金额（亿元）
- 流向趋势

**口径要求**: ETF 新闻净流入、季度/年度摘要和单日外推必须保留 `is_estimated=true`，不能通过手工改成 `false` 清除 gate。

### 1.4 融资融券 (Margin)

**数据源**:
- [上交所-融资融券](http://www.sse.com.cn/market/dealingdata/overview/margin/)
- [深交所-融资融券](http://www.szse.cn/market/dealingdata/rzrq/margin/)

**需要获取的数据**:
- 近5日融资融券余额变化（亿元）
- 近120日累计余额变化（亿元）
- 流向趋势

      "trend": "流入",
      "source": "Stage2.5 manual",
      "source_url": "https://data.eastmoney.com/etf/",
      "metric_basis": "news_net_flow",
      "window_evidence": "news_summary",
      "is_estimated": true,
      "note": "ETF新闻或季度摘要为 estimate-only，will not clear the gate"
    },
    "margin": {
      "recent_5d": 88.5,
      "total_120d": 1248.3,
      "trend": "流入",
      "source": "Stage2.5 manual",
      "source_url": "https://www.sse.com.cn/market/dealingdata/overview/margin/",
      "note": "数据来源: 上交所+深交所"
    }
  }
}
```

**关键要求**:
- `recent_5d` 和 `total_120d` 必须是**可解析的数字**，不能是描述性文本
- `trend` 使用 "流入" 或 "流出"
- `source` 标注为 "Stage2.5 manual"，且必须提供 `source_url`
- `source_url` 不等于窗口真实值；fund_flow 的 `source_tier` 由 URL/domain 推断，manual JSON 手填 `source_tier` 只作诊断，不能释放 gate
- `is_estimated=false` 仅适用于结构化来源直接覆盖目标窗口，且 `window_evidence` 为 `direct_window`、`direct_daily_series` 或 `direct_balance_delta`
- `news_net_flow`、`estimated_net_flow`、新闻/季度/年度摘要和单日外推必须保留 `is_estimated=true`，会被 gate 阻断或降级展示

## 步骤3: 运行注入脚本

  data/runs/${DATE}/market_data_stage2.json \
  data/runs/${DATE}/websearch_results_manual.json \
  data/runs/${DATE}/market_data_complete.json
```

## 注意事项

1. **数值格式**: `recent_5d`/`total_120d` 必须是数字（如 `132.6`），不能是文字（如 "波动"、"净流入"）
2. **单位统一**: 统一使用亿元
3. **更新后验证**: 注入后检查 `metadata.data_completeness` >= 0.8
4. **参考schema**: 详见 `src/datasource/models/market_data_contract.py`
