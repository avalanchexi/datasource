## Accepted Decisions

### D1: PR3 scope

Decision: Use canonical key registry + alias normalizer + compatibility read layer.

Do not rely on Pydantic `Field(alias=...)` for `monetary_policy` dict keys. The contract currently models `monetary_policy` as a dictionary, so field aliases do not normalize keys such as `reverse_repo_7d`, `mlf_rate`, `rrr`, or `tsf_growth`.

### D2: missing_items migration

Decision: Gradual convergence.

Use `metadata.missing_items` as the canonical source. Keep top-level `missing_items` and `gap_monitor` as compatibility views until replay tests prove old and new run shapes behave correctly.

### D3: PR5 scope

Decision: Downgrade PR5 to run_paths contract validation.

`src/datasource/utils/run_paths.py` already exists and is used by multiple stages. PR5 should add consistency tests and docs checks, not rebuild the path abstraction.

### D4: PR1 utils extraction

Decision: Semantic layering, not blanket DRY.

Only extract helpers with identical behavior. Keep separate contracts for strict JSON loading, optional JSON loading, normal numeric coercion, fund-flow amount parsing, normal placeholders, and legacy `7.13` placeholders.

### D5: pre-commit

Decision: Split pre-commit into a separate follow-up.

PR1 should remain behavior-preserving. `.pre-commit-config.yaml` is useful, but it should not be bundled with util extraction or repo-wide formatting.

### D6: PR verification

### D7: Pring golden tests
