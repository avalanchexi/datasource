# Test Plan

Generated by `/plan-eng-review` on 2026-04-27.

## Affected Flows

- CLI pipeline Stage2.5 -> Stage3 -> Stage4: replay fixed fixtures to verify JSON contracts, missing-data gates, and report generation stay stable after refactors.
- Pring analysis engine: verify score thresholds, final stage, confidence, and report-facing summaries remain unchanged before any module split.

## Key Interactions To Verify

- Run Stage2.5 injection with old alias keys, canonical keys, and mixed manual JSON keys, then verify the same canonical output shape.
- Run Stage3 with legacy top-level `missing_items`, `metadata.missing_items`, and clean canonical metadata, then verify blocking decisions are identical where intended.
- Run Stage4 with dated `gap_monitor.json` and explicit `--gap-monitor`, then verify pending/manual gaps block report generation and clean fixtures pass.

## Edge Cases

- Numeric placeholders: `None`, empty string, `N/A`, `0`, and legacy `7.13` must keep their distinct Stage2 and Stage2.5 semantics.
- JSON loading modes: strict readers must fail on missing or invalid input, optional diagnostic readers must return `None`.
- Monetary aliases: `reverse_repo_7d`, `mlf_rate`, `rrr`, `tsf_growth`, and canonical names must normalize without duplicate live entries.
- Pring score thresholds: each scoring function must cover boundary values on both sides of every threshold.

## Critical Paths

- Fixed fixture replay: `market_data_stage2.json` + `websearch_results_manual.json` -> `market_data_complete.json` -> `pring_result.json` -> Markdown report.
- Pring golden replay: copied fixture input from `data/runs/20260424` -> deterministic output comparison under `tests/fixtures/pring_golden/`.
- Release-only live smoke: run Stage1 -> Stage4 once after refactor PRs land. Do not use live Tavily/API output as per-PR byte-level diff gate.

## Required Test Files

- `tests/test_utils_coercion.py`: strict vs optional coercion and placeholder semantics.
- `tests/test_utils_json_io.py`: strict JSON load/save and optional diagnostic JSON load behavior.
- `tests/test_pring_scoring_golden.py`: score-level threshold boundaries and full-result golden checks.
- `tests/test_monetary_key_registry.py`: canonical key registry and conflict matrix.
- `tests/test_stage25_contract_replay.py`: Stage2.5 manual replay with old, new, and mixed keys.
- `tests/test_missing_items_compat.py`: metadata canonical source plus top-level/gap_monitor compatibility.
- `tests/test_run_paths_consistency.py`: Stage default output paths and explicit argument override behavior.

## Test-Safe IO Rule

Fixture replay must not mutate real `data/trend_history/min`. Stage2.5 should support `trend_history_base_dir=tmp_path` or an explicit `disable_trend_history_write` option before replay tests are added.
