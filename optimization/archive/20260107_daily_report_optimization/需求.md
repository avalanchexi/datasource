# 2026-01-07 日报生成复盘优化（需求草案）

## 背景
- Stage2 Tavily extract 422 导致自动抽取失败，多轮手工注入。
- 关键指标缺失（DXY/BDI/RRR/MLF）阻断 Stage3，反复补数成本高。
- 注入字段不匹配（ytd_change_pct 等）引发报告 N/A。
- 实时汇率使用过时估计值，方向判断错误。

## 目标
1) 建立数据质量指标体系并落库，形成每日趋势对比。
2) 提升可观测性：记录每个指标的获取耗时、来源、缺口原因、失败类型。
3) 策略自动化：将 422/缺口/异常值触发策略配置化，减少人工决策。
4) 来源分级与冲突解决：按权重合并并保留冲突日志。
5) 可重复运行与审计：输出运行参数快照，支持回溯。

## 范围
- Stage2/Stage2.5/Stage3/报告前的质量与观测增强。
- 新增质量指标、观测日志、冲突日志、运行快照的产出。
- 配置化策略与来源权重（不改变现有 pipeline 数据结构）。

## 非目标
- 不替换现有 Tavily/MCP/TuShare 数据来源策略。
- 不改变 Stage1/Stage3 的核心业务逻辑输出结构。
- 不引入新的外部依赖（除非必要且可控）。

## 需求拆解
## Stage2 Tavily 命中率低原因与优化要求
**问题复盘（与配置/流程强相关）**
- Tavily extract 422 达阈值后自动停用 extract，后续仅 search+regex，商品类/指数类命中率显著下降。
- 相关性评分过低时未触发过滤（低分全部保留），DeepSeek 在不相关内容上抽取，失败率升高。
- `preferred_domains` 作为硬性白名单 + `topic=news` 导致“报价类指标”命中新闻而非数值页面。
- `language=chinese` 但域名包含 Reuters/Yahoo 等英文站，语义匹配分数偏低。
- 发布机构校验过严（issuer/alias 未命中即 manual_required），即便抽到数值也被丢弃。
- 实时类统一 `time_range=day` + `days=2` + `max_age_days=1~3`，遇到周末/节假日容易过期。

**范围**
- 仅涉及 Stage2/Stage2.5 的搜索、抽取、校验与观测配置；不改动 Stage1/Stage3 输出结构。
- 优化 `search_profiles` 与 Stage2 策略逻辑（命中率/相关性/可解释性）。

**需求/优化点**
1) **search_profiles 调整**
   - 报价类指标（商品/ETF/指数）新增双语 query 或 ticker 关键词，`topic` 从 `news` 调整为更通用场景。
   - 域名白名单拆分为“官方报价/行情页优先 + 新闻源兜底”，避免只命中文章。
   - 对 USDCNH / BCOM / GSG 等补充更贴近报价页的 query 与 issuer_aliases。
   - 时间窗按指标粒度调整（如 USDCNH 可放宽 `days/max_age_days`）。
2) **搜索优化策略（命中率提升）**
   - **Query 结构标准化**：`中文指标名 + ticker/代码 + price/quote/level + 单位`，对报价类优先拼出“可解析数字”的关键词。
   - **多 query 兜底**：同一指标支持 2–3 条 query（行情页优先 → 官方公告 → 新闻兜底），按优先级执行。
   - **语言策略**：报价/指数类优先双语或英文关键词；宏观/政策类以中文为主。
   - **topic 精细化**：报价类使用更通用主题（非 news）；宏观类保留 news。
   - **域名策略**：区分“行情页优先域名”和“新闻兜底域名”，避免 Reuters/Yahoo 文章占满结果。
   - **排除域名（可选）**：对报价类允许配置 `exclude_domains`（屏蔽泛新闻/软广告站）。
   - **时效窗口分层**：实时类 `days=3~5`；宏观类 `time_range=month/quarter`，避免周末/节假日过期。
   - **抽取入口保护**：当 `score_max < 阈值` 时不进入抽取，直接标记 `manual_required` 并记录原因。
2) **低相关性保护**
   - 当全部结果 `score` 低于阈值时，不进入 DeepSeek 抽取，直接标记 `manual_required` 并记录原因。
   - 增加 `low_score_drop` 观测指标，便于复盘。
3) **extract 422 降级策略**
   - 422 触发应“按指标/短时窗口降级”，避免全局长期关闭 extract。
   - 记录 `tavily_extract_422` 发生指标清单，供后续手工补数与配置优化。
4) **发布机构校验放宽**
   - 当已抽到数值时，issuer 缺失仅作为提示，不强制 manual_required（可配置白名单）。
   - 对 USDCNH/MLF/逆回购等补充 alias 或降级为“提示”级别。
5) **可观测性补强**
   - 日志需明确：低分过滤原因、域名过滤原因、issuer 缺失原因、过期原因。
   - `logs/observability_${DATE}.json` 增加 `score_min/score_p50/score_p95` 指标。

**验收**
- 低相关性任务不会进入 DeepSeek（避免无效抽取）；
- extract 422 不再导致全局失效；
- 报价类指标命中率提升且可追溯原因；
- 关键失败原因在 observability/gap_monitor 中可定位。

## 资产层面 50 字结论生成（DeepSeek）
**目标**：对“非 TuShare 资产”（商品/外汇/债券/资金流向）基于当日变化给出不超过 50 字的中文结论，写入日报，失败可降级。

**范围与数据来源**
- 仅使用 `market_data_complete.json` 中的非 TuShare 资产数据：`commodities / forex / bonds / fund_flow`。
- 不从 `reports/*.md` 反向取数；不依赖历史报告内容。
- 字段优先级：`trend` > `ytd_change`/`change_120d` > `change_5d` > `daily_change`/`daily_change_pct`。

**生成逻辑（两步）**
1) 结构化摘要（程序内规则，避免空趋势）  
   - 按资产类别生成“方向 + 幅度”摘要；若趋势缺失，依据变化幅度推断（债券用 bp，其他用 %）。
2) DeepSeek 生成 50 字结论  
   - 输入为结构化摘要 + 统一提示词；输出强约束 `<= 50` 字。
   - 长度超限：截断并补句号；若输出为空则降级模板。

**失败兜底**
- DeepSeek 超时/失败：使用模板句（示例）  
  `资金流转正，汇率偏稳，债券小幅下行，商品分化。`
- 任何资产缺字段不影响整体输出，但需记录原因。

**集成点（推荐）**
- 报告生成前（`generate_simple_report*` 或 `tests/scripts/generate_simple_report_test.py`）生成并写入报告“资产层面结论”段落。
- 同步写入 `logs/observability_${DATE}.json`：输入摘要、输出文本、耗时、失败原因。

**验收与测试**
- 50 字以内硬性约束通过（包括中文标点）。
- DeepSeek 失败时不中断 Stage3/报告。
- 资产类缺失字段可降级，结论不为空。

## 120日趋势数据保留分析
**容易出问题的指标（120日趋势计算风险）**
- 宏观指标（如工业增加值/CPI/PPI/GDP/社融）：发布频率月度/季度，数据存在修订；用“日频 120日趋势”口径会失真。
- 货币政策利率（RRR/MLF/逆回购）：事件驱动、阶梯式变化；若按日做 120 日趋势，容易出现“长期为 0 变化”的假象，需基于事件日期计算。
- 资金流向（北向/南向/ETF/两融）：交易日不连续、港股/内地交易日差异、部分来源存在回补/修订；若缺失交易日或跨市场日历未对齐，会导致 120 日累计偏差。
- BDI 指数：个别日期缺失、时区/更新频率不稳定，且部分来源非官方；历史序列需要补齐与对齐。
- 商品期货（原油/铜/黄金等）：主力/连续合约切换、交割月变更，来源可能混用现货/期货；不统一合约口径会造成 120 日趋势突变。
- 外汇（USDCNY/USDCNH 等）：实时价 vs 收盘价混用、在岸/离岸口径差异；需要统一“日终收盘口径”。
- 债券收益率（US10Y/CN10Y/CDB10Y）：来源口径不同（盘中/收盘）、节假日缺口；若与其他日频数据混算，会导致趋势误差。

**适合长期保存用于趋势计算的指标（建议优先留存）**
- 日频市场数据（稳定口径）：主流股指收盘价（上证/深证/沪深300/恒指/标普500/纳指等）。
- 日频外汇（统一收盘口径）：USDCNY/USDCNH/EURUSD 等，需固定来源与收盘时间。
- 日频大宗商品（统一连续合约或现货口径）：黄金/原油/铜等，需固定符号与合约规则。
- 日频收益率（统一来源与时点）：US10Y/CN10Y/CDB10Y，建议固定为“收盘/日终”口径。
- 资金流向日度明细：北向/南向/ETF/两融的每日净流入；由此滚动计算 5d/120d 累计最稳妥（前提：交易日历对齐）。
  - 北向资金：当日值以新闻口径覆盖，但 5d/120d 仍以日度序列回算。
  - 南向资金：日度序列优先同花顺数据中心（无备份源）。

**建议存储口径（避免 120 日趋势偏差）**
- 日频指标统一使用“日终/收盘”值，避免与盘中值混用。
- 交易日历显式区分（A 股/港股/美股），资金流向需对齐对应市场日历。
- 宏观/政策类采用“事件序列”存储（发布日期 + 数值），趋势以“最近一次变动/过去 120 日是否变动”表述，而非日频斜率。
- 若需要 MA200/趋势指标（`above_ma200`/`ma50_slope`/`volatility_30d`），必须保留**至少 200 个交易日**收盘价；仅保留 120 日会导致指标失真或只能标记为 `is_estimated`。

**长期数据存储目录规划（新增 data 目录）**
- 新目录：`data/trend_history/`，用于保存可复用的“日级快照/事件序列”，支撑 120 日趋势计算。
- 建议存储内容（按类别分文件或单日汇总文件均可）：
  - 日频序列：股指收盘价、外汇收盘价、商品统一口径价格、债券收益率（收盘/日终）。
  - 资金流向：北向/南向/ETF/两融的**每日净流入**（滚动计算 5d/120d）。
  - 宏观/政策：按“事件序列”存储（发布日期 + 指标值 + 报告期），避免用日频补齐造成假趋势。
- 文件命名建议：`data/trend_history/YYYYMMDD_trend_snapshot.json`
- 最小字段建议：
  - `date`, `symbol`, `value`, `unit`, `source`, `source_timestamp`, `market_calendar`, `is_estimated`
  - 事件类额外：`report_period`, `release_date`, `event_type`
- 保留周期：建议 >= 400 天（含 MA200 计算缓冲），避免 120 日计算缺口。

## 120日数据缺口的计算可行性
**可用“周/月数据”近似的指标（可行但需标注估计）**
- 宏观指标：PPI/CPI/PMI/GDP/M0/M1/M2/TSF（本身是月度/季度发布，可用“近 4 个月差”近似 120 日变化）。
- 货币政策类：RRR/MLF/逆回购（事件驱动） → 以“最近一次变动 vs 120 天前”描述，建议标记 `is_estimated=True`。
- BDI：若日频难取，可用周度/周均值近似，但需标注“估计”。

**不建议用周/月数据近似的指标（误差大）**
- 商品/外汇/债券收益率/股指：波动高且日频变化显著，周/月数据会扭曲 120 日趋势。
- 资金流向：应以日度净流入累计为准；若仅周度可取，可做近似但需显著标注估计。

**结论**
- 慢频指标可用月度数据近似 120 日变化，但必须标注“估计/非日频”。
- 高频市场数据必须保留日度快照，避免用月度近似导致趋势失真。

## 非日报场景的连续性保障（方案A）
**方案A：轻量级日同步（不产出报告）**
- 每天只跑一个“同步任务”，写入 `trend_history`，不做 Stage3/报告。
- 同步内容仅包含：股指/外汇/商品/债券/资金流向的**日值**，以及宏观/政策**事件更新**。
- 优点：趋势序列连续、120 日计算稳定；缺点：仍需每天跑一个轻量任务。

**触发环节（推荐）**

## 2026-01-14/15 Tavily 搜索流程优化（实施记录）

### 问题复盘（今日 WebSearch 过程）

| 问题 | 根因 | 影响 |
|------|------|------|
| **Tavily extract 422** | API 配额限制 | 搜索成功但提取失败，value=null |

### 已实施修复

#### 阶段一：is_estimated 逻辑修复 ✅
**文件**：`inject_websearch_data_test.py`

**问题**：原逻辑 `entry['is_estimated'] = _coerce_bool(payload.get('is_estimated')) or bool(entry.get('is_estimated'))` 导致 `False or True = True`，手工注入的 `is_estimated: false` 无效。

**修复**：
```python
# 修复1：显式设置优先
if 'is_estimated' in payload:
    entry['is_estimated'] = _coerce_bool(payload.get('is_estimated'))
else:
    entry['is_estimated'] = bool(entry.get('is_estimated'))

# 修复2：fallback 不覆盖显式设置
if fallback_reason:
    if 'is_estimated' not in payload:  # 仅当未显式设置时
        entry['is_estimated'] = True
```

```python
def _post_injection_validation(market_data: Dict[str, Any]) -> None:
    """注入后校验，打印仍为估计值的字段。"""
    estimated_fields: List[str] = []
    # 检查 bonds, macro_indicators, monetary_policy, commodities, forex
    for bond in market_data.get('bonds', []) or []:
        if bond.get('is_estimated'):
            estimated_fields.append(f"bonds.{bond.get('name')}")
    # ... 其他类别

    print("\n[VALIDATION] 估计值校验:")
    if estimated_fields:
        print(f"  [WARN] 仍有 {len(estimated_fields)} 个估计值字段:")
        for field in estimated_fields:
            print(f"    - {field}")
    else:
        print("  [OK] 所有字段已去除估计值标记")
```

### 120日变化派生位置（方案1：Stage2.5 注入后派生）
**结论**：在 `inject_websearch_data_test.py` 注入完成后派生 120 日变化（最稳）。  
**原因**：此时数据最完整、来源已统一；补算后进入 Stage3/报告，避免报告阶段临时造数。  

**派生范围**
- 债券类：补算 `change_120d_bp`（如 CN10Y / CN10Y_CDB），优先从 `trend_history` 的 121 交易日序列首尾计算。
- 政策类：补算 `change_from_120d`（如 RRR/MLF/逆回购），优先从事件序列回看 120 日内前值；无前值则标注 `no_previous_value` 并提示“无前值可比”。

**输出要求**
- 写入 `market_data_complete.json` 后再进入 Stage3/报告。
- `gap_monitor`/`observability` 需记录派生失败原因（如 `trend_history_missing/no_previous_value`）。

### 国开债（CN10Y_CDB）计算页（基于 trend_history）
- **数据源**：`data/trend_history/min/series/bonds/CN10Y_CDB.json`（121 交易日）
- **窗口边界**
  - 最新值：2026-01-14，`value=1.96`
  - 5 日基准：2026-01-08，`value=2.2947`
  - 120 日基准：2025-07-17，`value=1.9644`
- **公式**
  - `change_5d_bp = (latest - base_5d) * 100`
  - `change_120d_bp = (latest - base_120d) * 100`
- **结果（bp）**
  - `change_5d_bp = -33.47`
  - `change_120d_bp = -0.44`
- **备注**：若基准值含 `is_estimated=true`（当前 120 日基准为估计），派生结果需同步标记估计或写入 `note` 说明。
