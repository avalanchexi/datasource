## 数据对齐需求
- 键名统一：历史 `market_data`/`missing_items`/`gap_monitor` 若含 `industrial_output`，需改为 `industrial`，否则 Stage2 任务缺模板且 Stage2.5 注入不会填补。
- Schema 保持：Stage2.5 注入仍按 fund_flow/commodities/forex/bonds 标准字段，确保 `metadata.data_completeness≥0.8` 后再进 Stage3。
- 流程一致性：Stage2 生成的缺口应通过 Stage2.5 `_manual/_auto.json` 注入后再跑 Stage3，避免报告出现 N/A。

## 数据对齐优化方案
1) 数据侧批量清洗（推荐）：将历史 `industrial_output` 全量重命名为 `industrial`（涵盖 `data/*market_data*.json`、`reports/gap_monitor_*.json` 等），避免 Stage2 生成空模板任务、Stage3 出现缺口。示例脚本：
   ```bash
   python3 - <<'PY'
   import json, pathlib
   def rename(obj):
       if isinstance(obj, dict):
           if 'industrial_output' in obj:
               obj['industrial'] = obj.pop('industrial_output')
           for v in obj.values(): rename(v)
       elif isinstance(obj, list):
           for v in obj: rename(v)
   for path in pathlib.Path("data").glob("*market_data*.json"):
       d = json.load(open(path, encoding="utf-8")); rename(d)
       open(path, "w", encoding="utf-8").write(json.dumps(d, ensure_ascii=False, indent=2))
       print("fixed", path)
   for path in pathlib.Path("reports").glob("gap_monitor_*.json"):
       d = json.load(open(path, encoding="utf-8")); rename(d)
       open(path, "w", encoding="utf-8").write(json.dumps(d, ensure_ascii=False, indent=2))
       print("fixed gap", path)
   PY
   ```
2) 代码侧容错（可选）：在 Stage2 入口加载 market_data 后做 alias 映射 `industrial_output→industrial`，同步修正 `missing_items`，确保旧文件不会生成无参数任务。
3) Stage2.5 前预检：运行前扫描文件确保无 `industrial_output`；注入后确认 `metadata.data_completeness≥0.8` 且 `missing_items` 为空。
4) Stage3 前断言（可选）：在 Stage3 启动脚本加断言，防止旧键流入分析。
