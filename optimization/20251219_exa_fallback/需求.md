# 2025-12-19 Exa SDK 作为 Tavily 失败任务兜底（需求草案）

## 背景
- 当前 Stage2 WebSearch 仅支持 Tavily；当 Tavily 配额耗尽或请求失败时，任务直接进入 `manual_required`，影响自动补数。
- 需要引入 Exa 作为 **仅失败任务** 的兜底搜索引擎，保持现有 Stage2 产物结构不变。
- 接入方式：**Python SDK（exa-py）**。

## 目标
1) Tavily 失败的任务可自动切换 Exa 搜索，减少缺口与人工补数。
2) 保持 `websearch_results` / `task_log` / `gap_monitor` 的结构与语义一致，便于审计回放。
3) 不改变资金流向数据约束：**northbound/southbound/etf/融资融券仍必须 MCP WebSearch**。

## 范围
- Stage2 Unified（`scripts/stage2_unified_enhancer.py`）的任务执行路径。
- 新增 Exa SDK 适配层（`src/datasource/adapters` 下）。
- 新增 `EXA_API_KEY` 环境变量与使用说明（不写入代码库）。

## 非目标
- 不替换 Tavily 为默认搜索引擎。
- 不改变 Stage1/Stage3 的数据来源策略。
- 不使用 Exa 填充 fund_flow（资金流向）与强制 MCP WebSearch 的指标。

## 现状流程理解（来自 AGENTS.md）
### 日常报表流水线
1) **Preflight**：校验 `.env` 中 `TAVILY_API_KEY/DEEPSEEK_API_KEY/TUSHARE_TOKEN`，清空代理。
2) **Stage1**：`scripts/stage1_data_collector.py` 产出原始市场数据（资金流向为占位）。
3) **Stage2**：`scripts/stage2_unified_enhancer.py` 执行 Tavily WebSearch + 抽取（DeepSeek/regex），同日 Tavily 只跑一次。
4) **Stage2.5**：`inject_websearch_data_test.py` 注入手工/自动 WebSearch 结果，产出 complete 数据。
5) **Stage3**：`scripts/stage3_pring_analyzer.py` 产出分析结果（可 `--allow-estimated`）。
6) **Report**：`tests/scripts/generate_simple_report_test.py` 生成报告。

### 关键约束
- 严禁从历史 `reports/*.md` 复用数据。
- fund_flow（北向/南向/ETF/融资融券）必须 MCP WebSearch；0/None 标记为 `异常零值-需核查`。
- Stage2.5 注入需满足 schema；`gap_monitor` 与 `missing_items` 必须清空后才可 Stage3/报告。

## 接入策略（仅失败任务使用 Exa）
### 触发条件
对每个任务：
1) Tavily 请求失败（异常、HTTP 429/402/403、返回空结果且多次重试失败）。
2) 当前任务非 fund_flow（northbound/southbound/etf/margin）。
3) Exa key 可用。

满足则触发 Exa 兜底；否则维持现有 `manual_required` 逻辑。

### Exa SDK 基本用法（示意）
```python
from exa_py import Exa

exa = Exa(api_key=os.getenv("EXA_API_KEY"))
resp = exa.search(
    query="美元 人民币 在岸 汇率 最新 报价",
    num_results=6,
    include_domains=["cfets.com.cn", "eastmoney.com"],
    start_published_date="2025-12-18",
    # 可选：contents={"text": True, "summary": True, "highlights": True}
)
results = resp.results
```

## 参数映射（Tavily → Exa SDK）
- `query` → `query`
- `max_results` → `num_results`
- `preferred_domains` → `include_domains`
- `max_age_days` → 计算 `start_published_date = today - max_age_days`
- `time_range/topic/language/search_depth/chunks_per_source/auto_parameters/days`：
  - Exa SDK无对应字段时**忽略**；必要时可拼入 `query`（如“最新/今日/最近7天”）

## Search Profiles 兼容性要求（Tavily & Exa）
目标：同一份 `search_profiles.py` 同时支持 Tavily 与 Exa 兜底，不改字段结构。

### 兼容原则
1) **关键词优先**：query 必须包含明确时间限定词（“今日/最新/收盘/公布/同比/公告”），以便 Exa 在缺少 `days/time_range` 参数时仍能命中最新内容。
2) **域名可复用**：`preferred_domains` 继续作为 Exa `include_domains`；不拆分、不新增 Exa 专用字段。
3) **时效兜底**：Exa 侧通过 `max_age_days` 计算 `start_published_date`（或 `start_crawl_date`）来替代 Tavily 的 `days/time_range`。
4) **数值型任务建议 keyword**：商品/汇率/债券/指数类在 Exa 兜底时使用 `type="keyword"`，减少“解读类文章”干扰。
5) **资金流向禁用 Exa**：northbound/southbound/etf/margin 仍按 MCP 规则执行，不触发 Exa。

### 不需要修改 profiles 的原因
- 当前 query 已普遍包含“今日/最新/公布”等强时间词，适配 Exa 语义/关键词搜索。
- `preferred_domains` 与 `max_age_days` 可直接用于 Exa 过滤与发布日期起点计算。

## 结果结构适配
需要将 Exa 结果映射为 Tavily 兼容片段，供 DeepSeek/regex 抽取复用：
```json
{
  "url": "...",
  "snippet": "...",   // 可用 highlights/summary
  "content": "...",   // 可用 text
  "score": null       // Exa无评分可置空
}
```
并在 `task_log` / `websearch_results` 中标记 `search_backend=exa`。

## 失败与降级策略
1) Exa 也失败 → 维持 `manual_required`。
2) Exa 返回空结果 → 直接 `manual_required`（不要反复重试）。
3) 任何情况下 fund_flow 仍保持 MCP/manual，不触发 Exa。

## 配置与安全
- `.env` 新增 `EXA_API_KEY`（不入库）。
- 可选：`EXA_BASE_URL`（如果 SDK 支持自定义端点）。
- 运行前建议在 preflight 中做 key 校验，但不强依赖。

## 日志与审计
在 `websearch_results` 增加：
- `search_backend`: "exa"
- `raw_results`: Exa 结果映射后的前三条片段
- `note`: `exa_fallback`/`exa_empty`/`exa_error:<reason>`

## 集成设计要点（落地点）
1) **Stage2 执行**：在 Tavily 搜索失败/空结果/配额类错误时，调用 Exa SDK 兜底；成功后继续走现有抽取逻辑。
2) **search_backend 标记**：Exa 兜底任务在 `task_log` 与 `websearch_results` 中标记 `search_backend=exa`，便于审计。
3) **保持 profiles 兼容**：不改 `search_profiles.py` 结构，仅在 Exa 调用侧使用 `max_age_days` 计算日期过滤，必要时用 `type="keyword"`。
4) **风险控制**：fund_flow 仍遵守 MCP 规则，不触发 Exa；Exa 失败则维持 `manual_required`，不重复重试。

3) fund_flow 仍遵循 MCP WebSearch 约束，不被 Exa 覆盖。
