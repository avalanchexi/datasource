# 背景扫描数据验证提示词模板

## 🔍 数据验证专用提示词

### 1. 完整验证提示词

```
请对这份背景扫描120日数据进行全面质量验证：

【验证维度】
1. 结构完整性(25%)：检查是否包含9个标准章节
2. 数据完整性(30%)：验证关键资产数据是否齐全
3. 数据合理性(35%)：检查价格、涨跌幅是否在正常范围
4. 时效性(10%)：确认数据日期的有效性

【必需章节清单】
✅ 市场结论要点
✅ 股票市场综述
✅ 商品与黄金
✅ 汇率变化
✅ 利率与债券收益率
✅ 资金流向综述
✅ 财经要闻
✅ 普林格阶段推断
✅ 附注说明

【关键数据检查】
- 股票指数：沪深300、上证50、创业板指、深证成指、上证指数、科创50
- 商品数据：WTI原油、Brent原油、COMEX铜、现货黄金、BCOM指数(GSG)
- 汇率数据：USD/CNY、USD/CNH、DXY
- 债券数据：CN10Y、US10Y、CN10Y_CDB

【输出格式】
请按以下格式输出验证结果：
1. 验证综合评分：X/100分
2. 各维度得分明细
3. 发现的问题列表
4. 改进建议
5. 是否可以直接生成日报的建议

评分标准：≥70分可生成日报，50-69分警告生成，<50分建议重新上传
```

### 2. 快速验证提示词

```
快速验证此背景扫描数据质量：
- 检查9个标准章节是否齐全
- 验证主要股票指数、商品、汇率、债券数据
- 评估价格数据合理性
- 给出0-100分评分及建议

如果≥70分，请说"验证通过，可生成日报"
如果<70分，请列出具体问题
```

### 3. 异常数据检测提示词

```
专门检查此背景扫描数据中的异常情况：

【重点检查】
1. 价格异常：ETF价格是否在0.5-50元合理范围
2. 涨跌幅异常：是否有超过±20%的异常波动
3. 数据缺失：哪些关键数据点显示为N/A或空值
4. 逻辑错误：普林格阶段判断与具体数据是否一致

【输出要求】
只报告发现的异常情况，如无异常则回复"数据检查正常"
对每个异常提供具体的数值和建议修正方案
```

## 📊 标准验证流程提示词

### 阶段1：结构检查

```
第一步：检查此背景扫描文件的章节结构
请确认以下9个章节是否完整存在：
1. 市场结论要点
2. 股票市场综述
3. 商品与黄金
4. 汇率变化
5. 利率与债券收益率
6. 资金流向综述
7. 财经要闻
8. 普林格阶段推断
9. 附注说明

输出格式：
- 完整章节：[列表]
- 缺失章节：[列表]
- 结构完整性评分：X/25分
```

### 阶段2：数据完整性检查

```
第二步：检查关键金融数据的完整性

【股票市场数据】请确认是否包含：
- 沪深300数据（价格、涨跌幅、技术指标）
- 上证50数据
- 创业板指数据
- 深证成指数据
- 上证指数数据
- 科创50数据

【商品市场数据】请确认是否包含：
- WTI原油数据
- Brent原油数据
- COMEX铜数据
- 现货黄金数据
- BCOM商品指数(GSG代理)数据

【汇率数据】请确认是否包含：
- USD/CNY数据
- USD/CNH数据
- DXY美元指数数据

【债券数据】请确认是否包含：
- CN10Y中国10年期国债收益率
- US10Y美国10年期国债收益率
- CN10Y_CDB中国10年期国开债收益率

输出：数据完整性评分：X/30分，并列出缺失的数据项
```

### 阶段3：合理性检查

```
第三步：数据合理性验证

请检查以下数据是否在合理范围内：

【价格合理性】
- 商品价格应与国际主流报价接近（WTI约60-120美元/桶、Brent约65-125美元/桶、COMEX铜约3-6美元/磅、现货黄金约1500-2500美元/盎司）
- 股票价格应在1-1000元范围内
- 汇率变动应在正常范围内

【涨跌幅合理性】
- 单日涨跌幅通常在±10%以内
- 超过±20%的变动需要特别说明

【技术指标合理性】
- RSI应在0-100范围内
- MACD数值应合理
- 均线排列应符合逻辑

【宏观数据合理性】
- PPI、CPI数据应在历史合理范围
- PMI数据应在0-100范围内
- 利率数据应符合当前市场环境

输出：合理性评分：X/35分，并列出任何异常数据
```

### 阶段4：时效性检查

```
第四步：数据时效性验证

请检查：
1. 数据日期是否明确标注
2. 数据是否过于陈旧（超过3个交易日）
3. 各部分数据日期是否一致
4. 是否为最新交易日的数据

输出：时效性评分：X/10分，并说明任何时效性问题
```

## 🎯 综合评估提示词

### 最终验证报告

```
基于以上四个阶段的验证结果，请生成综合评估报告：

【综合评分计算】
总分 = 结构完整性(25%) + 数据完整性(30%) + 合理性(35%) + 时效性(10%)

【评估结论】
- 总分≥70分：数据质量良好，可直接生成日报
- 50-69分：数据基本可用，生成日报时需加强风险提示
- <50分：数据质量不足，建议补充完善后重新上传

【报告格式】
# 背景扫描数据验证报告

## 验证概览
- 综合评分：X/100分
- 验证结论：[通过/警告/不通过]
- 验证时间：[当前时间]

## 分项评分
- 结构完整性：X/25分
- 数据完整性：X/30分
- 数据合理性：X/35分
- 时效性：X/10分

## 问题清单
[列出发现的具体问题]

## 改进建议
[提供具体的改进建议]

## 生成日报建议
[是否建议生成日报及注意事项]
```

## 🚨 异常情况处理

### 严重错误处理

```
如果发现以下严重问题，请立即报告：

【数据质量严重问题】
- 超过50%的关键数据缺失
- 多个价格数据明显异常
- 普林格阶段判断逻辑严重错误
- 数据时间跨度不一致

【处理建议】
对于严重问题，请回复：
"⚠️ 发现数据质量严重问题，不建议生成日报。
具体问题：[详细列表]
建议：请核实数据准确性后重新上传。"
```

### 轻微问题处理

```
如果发现轻微问题，可继续处理：

【可接受的轻微问题】
- 个别非关键数据缺失
- 少量价格数据超出常规范围但在可能范围内
- 格式不统一但不影响理解

【处理方式】
可以生成日报，但需在报告中标注：
"⚠️ 数据质量提醒：[具体问题]，分析时已考虑此限制。"
```

## 💡 最佳实践

### 验证前准备
1. 确保文件格式正确（MD/TXT）
2. 检查文件编码（推荐UTF-8）
3. 确认数据来源可靠性

### 验证过程中
1. 按阶段逐步验证，不要跳跃
2. 记录每个发现的问题
3. 区分严重问题和轻微问题

### 验证后处理
1. 根据评分决定是否生成日报
2. 对问题数据在报告中特别标注
3. 保存验证记录便于后续改进
